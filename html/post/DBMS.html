Database Management System (RookieDB)


RookieDB is a fully functional relational database engine built from scratch in Java as part of a semester-long course project at UC Berkeley (CS186). It covers core DBMS components including storage, indexing, query execution, join optimization, concurrency control, and crash recovery.

Key Features

  • Support for B+ Tree indexing with dynamic node splitting and merging.
  • Join processing via multiple algorithms: Simple Nested Loop Join (SNLJ), Block Nested Loop Join (BNLJ), and Grace Hash Join (GHJ).
  • Dynamic programming-based cost-based join planner for optimal query execution.
  • Multi-granularity locking and two-phase locking (2PL) for safe concurrency.
  • Crash recovery system using write-ahead logging, checkpoints, and rollback.

Tech Stack

  • Programming Language: Java
  • System Components: B+ Trees, Concurrency Control, Logging, Recovery
  • Algorithms: SNLJ, BNLJ, GHJ, Dynamic Programming (Join Planning)

Implementation

RookieDB is designed with modularity in mind, with separate components for storage management, indexing, and transaction processing. The B+ Tree implementation supports efficient inserts, deletes, and range queries. The query planner selects join strategies based on estimated cost using a dynamic programming algorithm. Concurrency is handled via multi-granularity 2PL, and crash recovery is enabled through logging and checkpoints.

Challenges and Solutions

  • Challenge: Supporting concurrent transactions without introducing deadlocks.
    Solution: Implemented multigranularity 2PL with waits-for graph cycle detection and rollback logic.
  • Challenge: Join performance across large datasets.
    Solution: Engineered and benchmarked multiple join algorithms and integrated a cost-based planner.
  • Challenge: Debugging tightly coupled subsystems.
    Solution: Refactored architecture for modularity and introduced integration testing at the subsystem level.

Results

  • Improved query performance by 80% with B+ Tree indexing and optimized join plans.
  • Increased transactional throughput by 70% after concurrency and recovery optimizations.
  • Handled complex transactions with crash recovery guarantees and modular system design.
