Dropbox Clone with End-to-End Encryption


This project is a secure file-sharing platform inspired by Dropbox, designed to safeguard user data privacy through end-to-end encryption. Files are encrypted on the client side, ensuring that the server has no access to unencrypted data.

Key Features

  • End-to-end encryption using RSA and AES-CTR for robust data protection.
  • Secure file sharing with revocation capabilities through hybrid encryption methods.
  • File management via linked list structures for dynamic updates and retrievals.
  • Data integrity verification using HMAC during all file operations.

Tech Stack

  • Programming Language: Golang
  • Security Protocols: RSA, AES-CTR, HMAC
  • Frameworks: Ginkgo (for testing)
  • System Design: Distributed Datastore, Hybrid Encryption

Implementation

The platform encrypts files on the client side using AES-CTR for encryption and RSA for secure key exchange. A distributed datastore securely stores encrypted metadata alongside file fragments. File sharing and access revocation are handled via hybrid encryption, ensuring that sensitive data remains inaccessible to unauthorized users.

Challenges and Solutions

  • Challenge: Securing user data against unauthorized server access.
    Solution: Implemented client-side encryption with RSA for secure key exchange.
  • Challenge: Enabling secure file sharing with access control.
    Solution: Designed hybrid encryption for sharing and revocation, ensuring privacy.

Results

  • Prevented unauthorized server access, achieving 100% data privacy.
  • Supported scalable file-sharing capabilities with reliable access control.
  • Conducted over 100 rigorous security tests, ensuring robustness and reliability.
