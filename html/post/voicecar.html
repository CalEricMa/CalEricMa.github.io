Voice Command Controlled Car


This project involved designing and implementing a voice-controlled robotic car capable of precise navigation using machine learning for voice classification and feedback control systems for trajectory correction. A strong emphasis was placed on data preprocessing, dimensionality reduction, and real-time classification accuracy to enable reliable and responsive control.

Key Features

  • Accurate voice classification using SVD and PCA for dimensionality reduction.
  • Implemented centroid-based clustering for robust command recognition.
  • Enhanced trajectory precision with closed-loop control for real-time path correction.
  • Integrated hardware and software systems for a seamless user experience.

Data Collection & Preprocessing

Voice data was collected using a microphone setup, sampling commands under varied conditions. Preprocessing steps included:

  • Enveloping: Detected the amplitude envelope of audio signals to isolate key features of the voice commands.
  • Normalization: Scaled audio signals to a uniform range to ensure consistency across samples.
  • Demeaning: Subtracted the mean from each feature dimension, centering the data around the origin for PCA.

Dimensionality Reduction with SVD and PCA

To reduce the high dimensionality of the audio data:

  • Singular Value Decomposition (SVD): Decomposed the data matrix into orthogonal components to extract key features efficiently.
  • Principal Component Analysis (PCA): Identified the top three principal components that captured the most variance, significantly reducing computational complexity while retaining essential information.
  • The reduced dataset was projected into the PCA basis, enabling effective visualization and clustering.

Clustering and Classification

Centroid-based clustering was employed to classify the commands:

  • For each command, centroids were computed by averaging feature vectors in the reduced PCA space.
  • Test samples were classified by calculating the Euclidean distance to each centroid, assigning the closest one.
  • Achieved 100% accuracy on test samples, demonstrating the reliability of the clustering method.

Model Enhancement

To further improve classification accuracy:

  • Outliers were removed from the training dataset to reduce noise in the clustering process.
  • Experimented with different numbers of principal components to balance accuracy and computational efficiency.
  • Introduced an additional noise filtering step to handle environmental disturbances during real-time classification.

Car Control Integration

The classified commands controlled the car as follows:

  • "Congrats": Drive straight (long).
  • "Stop": Drive straight (short).
  • "Unicorn": Turn left.
  • "Roni": Turn right.

The closed-loop control system used encoder feedback to correct deviations from the intended trajectory, ensuring precise execution of commands.

Challenges and Solutions

  • Challenge: Handling variability in voice samples.
    Solution: Applied normalization and SVD to extract consistent features across samples.
  • Challenge: Noise interference during real-time classification.
    Solution: Integrated noise filtering and enhanced centroid computation for robustness.
  • Challenge: Maintaining stability in car control.
    Solution: Implemented a feedback-based closed-loop system with error correction.

Results

  • Classification Accuracy: Achieved 100% accuracy on test samples with reliable real-time performance.
  • Trajectory Precision: Closed-loop control reduced path deviations and maintained stable motion.
  • Integration Success: Seamless coordination between classification and car control systems.
